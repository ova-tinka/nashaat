Use case Id: UC04    Manage Workout Plan (CRUD)

**Brief Description**
An authenticated user creates, views, edits, and deletes workout plans. Plans can be created manually (exercises, sets, reps, duration) or generated by AI (VIP only). Plans are associated with days of the week and support workout logging and screen-time earning.

**Primary actors**
Authenticated User

**Preconditions:**
1. User is authenticated and has completed onboarding (weekly exercise target set).
2. Device has an active internet connection.

**Post-conditions:**
- Create: New plan in workout_plans table; appears in list.
- Edit: Plan and exercises updated; future logs use updated details.
- Delete: Plan soft-deleted and removed from active list; historical logs preserved.

**Alternative flows:**
4a. Exercise not in library: subsystem offers "Create Custom Exercise"; user defines and it is saved for their use.
4b. Plan name already exists: subsystem prompts to choose a different name.
4c. No exercises added: subsystem prompts that at least one exercise is required.
6a. Free user attempts AI plan: subsystem shows VIP upsell and links to Subscription Management.
6b. AI generation fails: subsystem prompts to try again later or create manually.
6c. AI regeneration limit (3) reached: Regenerate disabled; user can edit manually.
10a. Editing plan in use today: subsystem warns that changes apply to today's workout.
12a. Deleting only active plan: subsystem warns and suggests creating a new plan first.
